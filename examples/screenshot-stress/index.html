case 2

Same-origin <img> (the happy path)

Baseline — html-to-image embeds same-origin images cleanly. If this fails, something fundamental is broken.

case 3

CORS-friendly cross-origin <img>

Picsum sends access-control-allow-origin: *. html-to-image's fetch+embed should resolve it. If this is blank, the embed step is failing.

case 4

CSS background-image (not <img>)

html-to-image's embedImages only handles <img>. Background images need inline-style serialization with data-URLs; common silent blank-fail mode on CMS-hosted backgrounds.

case 5

<video> element

Same-origin video — html-to-image can't extract its current frame. Should trigger the hasVideo tripwire in assessCaptureQuality → layer-2 escalation.

case 6

<canvas> with live drawing

Browser security may block readback. Should trip the hasCanvas tripwire.

case 7

Animated background (CSS keyframes)

Captured mid-paint. The blank-detection heuristic should recognize uniform color and escalate. Repeat picks to see if the captured color drifts.

PULSING
case 8

Element inside Shadow DOM (open)

html-to-image walks the document tree but not into shadowRoots — children may not render in the rasterise. captureDiagnostics.shadowDomDepthInCrop should be > 0 here.

case 9

Cross-origin <iframe>

Iframes are an opaque rectangle to html-to-image. captureDiagnostics.hasIframeInCrop should be true. Layer-2 (tab-share) is the only way to capture iframe pixels.

case 11

Element inside transform: scale() parent

Transformed parents can confuse the crop coordinate math — element bbox doesn't account for the transform. Crop may miss the visual position.
